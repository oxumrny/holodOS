# holodOS

Личный веб-трекер продуктов в холодильнике. Не публикуется в App Store — для себя и близких (публикация в интернет пока не сделана).

---

## Стек

| Слой | Технология |
|------|------------|
| Фронтенд | Vite 6 + React 19 + TypeScript |
| Бэкенд / БД | Supabase (PostgreSQL + REST API) |
| Стили | обычный CSS (файлы рядом с компонентами) |

Раньше проект был **Expo + React Native**; переделан на веб по запросу пользователя.

## Как устроено приложение

Две основные вкладки:

| Вкладка | `status` в БД | Назначение |
|---------|---------------|------------|
| **В холодосе** | `active` | продукты, которые есть дома |
| **Список покупок** | `finished` | продукты, которые закончились и нужно купить |

**Настройки** (кнопка справа вверху) — управление категориями и смена категории у продуктов.

### Действия с продуктом

- **Добавить** — форма на вкладке «В холодосе»; название сохраняется в **нижнем регистре**
- **+** (зелёный плюс слева) — продукт закончился → перенос в список покупок (`finished`)
- **Чекбокс** (в списке покупок) — отметить «куплено» → вернуть в холодос (`active`)
- **×** (красный крестик справа) — удалить навсегда из базы
- **Дубликаты** — нельзя добавить продукт с тем же именем (без учёта регистра); сообщение «Такой продукт уже есть»

### Категории

- При добавлении категория определяется автоматически по ключевым словам (`src/lib/detectCategory.ts`)
- На главных вкладках — фильтр по категориям; сортировка по категориям, без визуальных блоков-разделителей
- В **Настройках** — кастомные категории, переименование, скрытие, смена категории у продукта
- Конфиг категорий хранится в **localStorage** (`src/lib/categoryConfig.ts`), не в Supabase

```

    Settings.tsx          — настройки категорий и продуктов
    ErrorBanner.tsx       — ошибки видны в браузере (не только в консоли)
  hooks/
    useProducts.ts        — CRUD: загрузка, добавление, finished, restore, delete
  lib/
    supabase.ts           — клиент Supabase + проверка .env
    detectCategory.ts     — авто-категория по названию
    categoryConfig.ts     — localStorage: кастом/скрытые/алиасы категорий
  types/
    product.ts            — Product, ProductStatus, CATEGORY_ORDER
supabase/
  schema.sql              — схема таблицы + RLS-политики
.env.example              — шаблон переменных окружения
ROADMAP.md                — идеи будущих фич (low-stock, срок годности, рецепты…)
```

### Архитектурные особенности

- Два независимых экземпляра `useProducts('active')` и `useProducts('finished')` в `App.tsx`
- После **удаления** обновляются **обе** вкладки (`handleDelete` в App)
- При переключении вкладки вызывается `refresh()` соответствующего хука
- Удаление проверяет, что строка реально удалилась (`.delete().select('id')`)

## База данных (Supabase)

Таблица `products`:

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | uuid | первичный ключ |
| `name` | text | название (в нижнем регистре) |
| `status` | text | `active` или `finished` |
| `category` | text | категория, по умолчанию `прочее` |
| `created_at` | timestamptz | когда добавили |
| `finished_at` | timestamptz | когда перенесли в список покупок |

Скрипт: `supabase/schema.sql`. Если таблица создана раньше:

```sql
alter table products add column if not exists category text not null default 'прочее';
```

RLS включён; политики разрешают read/insert/update/delete всем (личный проект без авторизации).

## Запуск

### 1. Переменные окружения

```bash
cp .env.example .env
```

Заполнить в `.env`:

```
VITE_SUPABASE_URL=https://ваш-проект.supabase.co
VITE_SUPABASE_ANON_KEY=ваш-anon-key
```

Ключи: Supabase → **Project Settings → API** (нужен **anon public**, не `service_role`).

### 2. Установка и dev-сервер

```bash
npm install
npm run dev
```

Открыть `http://localhost:5173`

### 3. С телефона в той же Wi‑Fi

```bash
npm run dev -- --host
```

В терминале появится **Network**-адрес (например `http://192.168.1.42:5173`) — открыть в Safari.

### 4. Сборка

```bash
npm run build
npm run preview
```

## Частые проблемы

| Симптом | Что проверить |
|---------|---------------|
| `ERR_CONNECTION_REFUSED` на :5173 | не запущен `npm run dev` |
| «Не удалось загрузить данные» / invalid API key | `.env`, ключи Supabase, перезапуск dev-сервера |
| Продукт в базе есть, в списке нет | вкладка «В холодосе» (не «Список покупок»); фильтр **«Все»**; в БД `status = active` |
| «Не удалось удалить продукт» | в Supabase нет delete-политики — выполнить блок из `schema.sql` |
| После удаления нельзя добавить снова | старая запись с тем же именем всё ещё в БД; проверить Table Editor |

1. **Старт** — мобильное приложение Expo + React Native + Supabase

4. **Функции** — категории, дубликаты, нижний регистр, удаление
5. **UI-итерации** (отдельный чат) — чекбоксы, крестик вместо «Удалить», зелёный «+», страница Настройки, редактирование категорий, ошибки в UI, refresh при смене вкладки
